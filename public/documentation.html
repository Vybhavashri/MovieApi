Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

REQUEST URL HTTP METHDOD REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
Return the home page of myFlix App / GET None Message saying 'Welcome to myFlix App'
Return a list of ALL movies to the user /movies GET None Return list of all the movies
Return data about a single movie by title to the user /movies/:title GET None A JSON object holding data about a single movie.
Example:
{ movieID: 1, title: "The Shawshank Redemption", rank: "1", genre: "Drama", director: "Frank Darabont", yea: "1994", duration: "2h 22m" }
Return the list of all movies by genre to the user /movies/genre/:genre GET None A JSON object holding list of movies of that grene
Example:
{ "movieID": 6, "title": "The Lord of the Rings: The Return of the King", "rank": "6", "genre": "Adventure", "director": "Peter Jackson", "year": "2003", "duration": "3h 21m" }, { "movieID": 9, "title": "The Lord of the Rings: The Fellowship of the Ring", "rank": "9", "genre": "Adventure", "director": "Peter Jackson", "year": "2001", "duration": "2h 58m" }
Return data about a director by name /directors/:name GET None A JSON object holding data of a director
Example:
{ "directorID": 5, "name": "Frank Darabont", "description": "Writer, Producer, Writer. Three-time Oscar nominee", "birthDate": "28/01/1959", "deathDate": "" }
Allow new users to register /users POST A JSON object holding data about the new user
Example:
{ "name": "User5", "emailID": "user5@mydomain.com", "password": "user5" } 		A text indicating a new user is registered.
Allow users to update their user info (email) /users/:name/:emailID PUT A JSON object holding the EmailID.
Example:
{ "emailID": "newuser1@mydomain.com" } 		A text indicating User emailID was changed.
Allow users to add a movie to their list of favorites /users/:name/:title POST None A text indicating Movie has been added to your favorites list.
Allow users to remove a movie from their list of favorites /users/:name/:title DELETE None A text indicating Movie has been deleted from favorites list.
Allow existing users to deregister /delete/:name DELETE None A text message indicating the user successfully deregistered.
